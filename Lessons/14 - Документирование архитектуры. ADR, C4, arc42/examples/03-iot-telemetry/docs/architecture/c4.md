# C4 — Платформа IoT-телеметрии

> Уровни не смешиваются: на Context — только система, акторы и внешние системы; на Container — деплоимые единицы; на Component раскрыт один контейнер.

## Level 1 — System Context

```mermaid
C4Context
    title System Context — Платформа IoT-телеметрии

    Person(operator, "Оператор мониторинга", "Дашборды, реакция на алерты")
    Person(engineer, "Инженер / аналитик", "Исторический анализ, настройка порогов")
    System_Ext(gateway, "Edge Gateway", "Узел на объекте: агрегирует датчики, буферизует, публикует телеметрию по MQTT")

    System(platform, "Платформа IoT-телеметрии", "Приём, обработка, хранение телеметрии и алертинг по порогам")

    System_Ext(notify, "Notification провайдер", "Email / SMS / мессенджеры")
    System_Ext(incident, "Incident / CRM система", "Тикеты по критичным алертам")
    System_Ext(bi, "BI / DWH", "Отчётность поверх агрегатов")

    Rel(gateway, platform, "Публикует телеметрию", "MQTT/TLS")
    Rel(operator, platform, "Смотрит метрики, получает алерты", "HTTPS")
    Rel(engineer, platform, "Запросы истории, настройка правил", "HTTPS")
    Rel(platform, notify, "Отправляет оповещения", "HTTPS")
    Rel(platform, incident, "Заводит тикеты", "HTTPS")
    Rel(platform, bi, "Выгружает агрегаты", "JDBC/экспорт")
```

**Граница системы:** всё внутри «Платформы IoT-телеметрии» — наше. Edge Gateway показан как внешняя система: его прошивку/агента мы не разрабатываем, у нас с ним только контракт MQTT (см. [ADR-0001](../adr/0001-mqtt-edge-gateway.md)). Notification, Incident/CRM и BI — внешние интеграции.

## Level 2 — Container

```mermaid
C4Container
    title Container — Платформа IoT-телеметрии

    System_Ext(gateway, "Edge Gateway", "MQTT-клиент на объекте")
    Person(operator, "Оператор мониторинга")
    Person(engineer, "Инженер / аналитик")

    System_Boundary(platform, "Платформа IoT-телеметрии") {
        Container(broker, "MQTT Broker", "EMQX / VerneMQ", "Терминация MQTT/TLS, аутентификация шлюзов, буфер на входе")
        Container(bridge, "Ingestion Bridge", "Go", "Мост MQTT → Kafka: разбор топиков, бэкпрешер")
        ContainerQueue(kafka, "Event Log", "Kafka", "Партиционированный лог сырой телеметрии")
        Container(processing, "Ingestion/Processing Service", "Java + Flink", "Валидация, обогащение, оконные агрегаты, правила алертинга")
        ContainerDb(tsdb, "Time-series DB", "TimescaleDB", "Сырьё + агрегаты, ретеншн, даунсемплинг")
        Container(alertSvc, "Alert/Notification Service", "Go", "Дедупликация, эскалация, доставка нотификаций")
        Container(deviceSvc, "Device Registry", "Go", "Каталог устройств, provisioning, пороги")
        ContainerDb(metaDb, "Metadata DB", "PostgreSQL", "Устройства, шлюзы, правила, пороги")
        Container(queryApi, "Query API", "Go", "REST/gRPC: история, агрегаты, статусы для дашбордов")
        Container(ui, "Web Dashboard", "SPA", "Графики, карты алертов, настройка правил")
    }

    System_Ext(notify, "Notification провайдер")
    System_Ext(incident, "Incident / CRM система")
    System_Ext(bi, "BI / DWH")

    Rel(gateway, broker, "Публикует телеметрию", "MQTT/TLS")
    Rel(broker, bridge, "Доставляет сообщения", "MQTT sub")
    Rel(bridge, kafka, "Пишет сырьё", "Produce")
    Rel(processing, kafka, "Читает поток", "Consume")
    Rel(processing, deviceSvc, "Запрашивает метаданные/пороги", "gRPC")
    Rel(processing, tsdb, "Пишет точки и агрегаты", "SQL/COPY")
    Rel(processing, alertSvc, "Эмитит сработавшие алерты", "Kafka topic")
    Rel(alertSvc, notify, "Доставляет оповещения", "HTTPS")
    Rel(alertSvc, incident, "Заводит тикеты", "HTTPS")
    Rel(alertSvc, metaDb, "Состояние алертов", "SQL")
    Rel(deviceSvc, metaDb, "Читает/пишет", "SQL")
    Rel(operator, ui, "Использует")
    Rel(engineer, ui, "Использует")
    Rel(ui, queryApi, "REST/HTTPS")
    Rel(queryApi, tsdb, "Запросы истории/агрегатов", "SQL")
    Rel(queryApi, metaDb, "Метаданные устройств", "SQL")
    Rel(bi, tsdb, "Выгрузка агрегатов", "JDBC")
```

Каждый контейнер — отдельно деплоимая и масштабируемая единица (это **не** Docker-контейнер). Поток приёма (`broker → bridge → kafka → processing → tsdb`) и тракт чтения (`queryApi → tsdb`) физически разделены.

## Level 3 — Component (контейнер «Ingestion/Processing Service»)

```mermaid
C4Component
    title Component — Ingestion/Processing Service

    ContainerQueue(kafka, "Event Log", "Kafka")
    Container(deviceSvc, "Device Registry", "Go")
    ContainerDb(tsdb, "Time-series DB", "TimescaleDB")
    Container(alertSvc, "Alert/Notification Service", "Go")

    Container_Boundary(processing, "Ingestion/Processing Service") {
        Component(consumer, "Stream Consumer", "Flink source", "Читает партиции Kafka, фиксирует offset/checkpoint")
        Component(validator, "Validator", "Domain", "Схема, диапазоны, отбрасывание/карантин битых точек")
        Component(enricher, "Enricher", "Domain", "Привязка к устройству, единицы измерения, метки объекта")
        Component(metaCache, "Metadata Cache", "Caching", "Локальный кэш устройств и порогов из Device Registry")
        Component(window, "Windowing Aggregator", "Flink operator", "Скользящие/тумблинговые окна: avg, min, max, rate")
        Component(ruleEngine, "Rule Evaluator", "Domain", "Сравнение значений и агрегатов с порогами правил")
        Component(tsWriter, "TSDB Writer", "Sink", "Батч-запись сырья и агрегатов в TimescaleDB")
        Component(alertEmitter, "Alert Emitter", "Sink", "Публикация событий алертов в Kafka-топик")
    }

    Rel(consumer, validator, "Сырые точки")
    Rel(validator, enricher, "Валидные точки")
    Rel(enricher, metaCache, "Берёт метаданные/пороги")
    Rel(metaCache, deviceSvc, "Подгружает при промахе", "gRPC")
    Rel(enricher, window, "Обогащённые точки")
    Rel(enricher, tsWriter, "Сырьё на запись")
    Rel(window, ruleEngine, "Агрегаты окон")
    Rel(enricher, ruleEngine, "Мгновенные значения")
    Rel(ruleEngine, alertEmitter, "Сработавшие условия")
    Rel(window, tsWriter, "Агрегаты на запись")
    Rel(tsWriter, tsdb, "COPY/batch insert")
    Rel(alertEmitter, alertSvc, "Alert events", "Kafka")
```

**Идея компонентов:** приём и обработка идут одним конвейером поверх Flink — это даёт exactly-once запись и единое место для оконных агрегатов и правил (см. [ADR-0003](../adr/0003-stream-processing.md), [ADR-0004](../adr/0004-alerting-rules.md)). `Metadata Cache` снимает нагрузку с Device Registry: пороги читаются на каждом сообщении, но меняются редко.
